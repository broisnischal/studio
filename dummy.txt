Cause (Arch-specific)
Tauri runs linuxdeploy, which bundles an old strip tool. Arch’s system libraries use newer ELF sections (.relr.dyn) that that strip does not understand, so you get errors like:

strip: unknown type [0x13] section `.relr.dyn'
linuxdeploy treats that as fatal → failed to run linuxdeploy.

CI on Ubuntu 22.04 usually works because those libs are built with an older toolchain.

Fixes
1. Build AppImage on Arch (recommended if you want AppImage)
NO_STRIP=1 npm run tauri build
Or use the script added in the repo:

npm run tauri:build:arch
NO_STRIP=1 skips the broken strip step. The AppImage is a bit larger but works.

2. Skip AppImage on Arch (simplest)
You already have a working package:

npm run tauri build -- --bundles deb
Artifacts:

src-tauri/target/release/bundle/deb/db-studio_0.1.2_amd64.deb
Or run directly: ./src-tauri/target/release/app
On Arch, .deb is for Debian/Ubuntu (dpkg -i). For daily use on Arch, the binary or AppImage is more natural.

3. Optional: FUSE for running AppImages
If the AppImage builds but won’t run:

sudo pacman -S fuse2
Or:

./db-studio_0.1.2_amd64.AppImage --appimage-extract-and-run
